# ab_behave コマンドセンター型リデザイン

## 概要

Sony aibo PWA リモコン「ab_behave」の UI を、従来の縦スクロール＋タブ切替構成から「コマンドセンター型」のワンビュー・ダッシュボードに破壊的変更する。ステータスをヘッダーにコンパクトに統合し、画面の大部分をアクション操作に割り当てる。

## 設計方針

- **操作特化**: 情報取得とアクションのタブ分離を廃止し、すべてを1画面に統合
- **aibo中心**: ステータス（バッテリー・眠さ）を常時ヘッダーに表示し、aiboの状態を常に把握可能
- **M3 Expressive 継続**: 既に適用済みの Material 3 Expressive デザイントークン（カラー・シェイプ・モーション）を引き継ぐ
- **フレームワークなし**: 従来通り Pure HTML/CSS/JS で実装

## コンポーネント構成

### 1. ヘッダー

**モバイル:**
- 左: アプリタイトル「🐕 ab_behave」
- 右: ステータスチップ（バッテリー・眠さ）横並び ＋ 歯車アイコン（設定トグル）
- ステータスチップはバッテリー/眠さ取得後に動的に更新される
- 初期状態（未取得時）は「🔋 --」「😪 --」のプレースホルダーチップを表示（グレートーン）

**PC (1024px+):**
- ヘッダーはなくなり、これらの要素は左サイドバーに移動

### 2. 設定パネル

**モバイル:**
- ヘッダー右の歯車アイコンをタップするとヘッダー下にスライドダウンで展開
- 中身は既存と同じ: トークン入力、保存ボタン、接続テストボタン、デバイス情報
- パネル外タップまたは再度歯車タップで閉じる

**PC:**
- 左サイドバーの最下部にアコーディオン形式で配置
- 「⚙️ 設定」ヘッダーをクリックで展開/折りたたみ
- 折りたたみ時は「接続中: [デバイス名]」をコンパクトに表示

### 3. モード切替ボタン

- クイックアクションバーの**上**に全幅で配置
- 幅はクイックアクションバーと同一
- 既存のトグル動作を維持（NORMAL ↔ DEVELOPMENT）
- スタイル: M3 Filled Button（standby = secondary, normal = tertiary）

### 4. クイックアクションバー

- 固定プリセット4つ: 立つ(stand)・すわる(sit)・伏せる(down)・ダンス(dance)
- 横並びのアイコン＋ラベル形式
- 背景: `surface-container-high` で区切られたバー
- 各アイコンは白背景の角丸カード内に配置
- タップで即座に API 実行（姿勢変更 or play_motion）

### 5. カテゴリリスト（すべてのアクション）

「すべてのアクション」ラベルの下に、4つのカテゴリをカード形式で縦に並べる:

1. **🐾 姿勢を変える** — `change_posture` API（10種）
2. **🎮 ふるまい** — `play_motion` API（7サブカテゴリ、それぞれアコーディオン）
3. **🎪 トリック** — `play_trick` API（7種）
4. **🐕 首を動かす** — `move_head` API（5プリセット＋カスタムスライダー）

各カテゴリカード:
- 閉じた状態: カテゴリ名 ＋ 件数 ＋ ▼ アイコン
- 開いた状態: カード内部にアクションボタンのグリッドが展開（インライン展開）
- 「ふるまい」のみ2段階: カテゴリカード展開 → サブカテゴリアコーディオン → アクションボタン

### 6. ステータス取得の統合

従来の「情報取得」タブを廃止。ステータスの取得方法:
- ヘッダーのステータスチップをタップすると、対応する API を実行して更新
- バッテリーチップタップ → `hungry_status` 取得 → チップ更新
- 眠さチップタップ → `sleepy_status` 取得 → チップ更新
- 取得中はチップにローディング表示（スピナーまたはパルスアニメーション）
- 未取得時は「🔋 --」「😪 --」のようなプレースホルダー表示

## 変更されるファイル

| ファイル | 変更内容 |
|---|---|
| `index.html` | HTML 構造の大幅変更（タブ廃止、ヘッダー再構成、サイドバー追加） |
| `style.css` | CSS 全面書き換え（コマンドセンターレイアウト、レスポンシブ） |
| `script.js` | タブ切替の削除、ステータスチップのタップ取得、設定パネル制御の変更 |
| `sw.js` | キャッシュバージョン更新 |

## 変更されないもの

- API 通信ロジック（`executeMotion`, `executeTrick`, `executeMoveHead`, `executePosture` 等）
- 暗号化ユーティリティ（AES-GCM）
- データ定義（`MOTION_GROUPS`, `TRICK_LIST`, `MOVE_HEAD_PRESETS`）
- Service Worker のキャッシュ戦略
- PWA マニフェストの基本構造

## 動作フロー

### 初回アクセス（未設定時）

1. ヘッダー表示（ステータスチップは非表示）
2. 設定パネルが自動展開
3. メインエリアにセットアップ案内を表示
4. トークン保存 → 接続テスト → デバイス接続完了
5. 設定パネルが閉じ、コマンドセンターが表示される

### 通常利用

1. ヘッダーのステータスチップをタップ → バッテリー/眠さ取得・更新
2. モード切替ボタンで NORMAL ↔ DEVELOPMENT
3. クイックアクションバーで頻用アクションを即実行
4. カテゴリカードを展開して詳細アクションを選択・実行

## エラーハンドリング

- API エラー: 各アクション実行後にステータスメッセージ表示（既存の `showStatus` を継続利用）
- ステータスメッセージの表示位置: メインエリア下部に既存の固定位置表示を継続
- ステータスチップ取得失敗時: チップを error カラーに変更し、タップで再取得可能
